is_valid()[source]¶

Returns whether or not the memo is valid.

It should be noted that there are no guarantees if the memo is valid, only if the memo is invalid. That is, this function may return false positives.

Stricter validation can be done via AgoraMemo.is_valid_strict(). However, AgoraMemo.is_valid_strict() is not as forward compatible.

agora.model.payment module¶

class agora.model.payment.Payment(sender, destination, payment_type, quarks, source=None, invoice=None, memo=None)[source]¶

Bases: object

The Payment object, which represents a payment that will get submitted.

Parameters

  • sender (PrivateKey) – The PrivateKey <agora.model.keys.PrivateKey of the account from which funds will be sent.

  • destination (PublicKey) – The PublicKey <agora.model.keys.PublicKey of the account to which funds will be sent.

  • payment_type (TransactionType) – The TransactionType of this payment.

  • quarks (int) – The amount being sent.

  • source (Optional[PrivateKey]) –

    (optional) The PrivateKey <agora.model.keys.PrivateKey of the account that will act as the source of the transaction. If unset, the sender will be used as the transaction source.

    On Stellar, this is where the transaction fee and sequence number is taken/chosen from.

    On Solana, this is where the fee is taken from.

  • invoice (Optional[Invoice]) – (optional) An Invoice to associate with this payment. Only one of invoice or memo should be set.

  • memo (Optional[str]) – (optional) The text memo to include with the transaction. Only one of invoice or memo should be set.

class agora.model.result.EarnResult(earn, tx_hash=None, error=None)[source]¶

Bases: object

The EarnResult object, which contains the result of a submitted earn.

Parameters

  • earn (Earn) – The originally submitted earn.

  • tx_hash (Optional[bytes]) – (optional) The hash of the transaction that was submitted, if one was submitted for this earn. If omitted, it indicates that no transaction was submitted for this earn.

  • error (Optional[Exception]) – (optional) An Exception indicating why the earn failed. The absence of an error does not indicate that the earn was submitted successfully, only that if it failed, either its transaction failed due to another earn, or it was not submitted at all.
